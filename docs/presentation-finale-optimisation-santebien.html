CI/CD vert Automatisation

Pipeline Green CI orienté modèle

Limiter les runs

Déclenchement par chemins, annulation des runs obsolètes et cache pip.

Mesurer le modèle

`scripts.measure_ai_model` produit taille, accuracy, latence, CO₂ et accord FP32/INT8.

Bloquer les régressions

`scripts.green_gates` bloque si le modèle devient plus lourd, moins fiable ou trop lent.

L'API et le cache restent mesurés comme contexte, mais la décision principale du pipeline repose sur le modèle.

Ce qu'on dit au jury

Notre projet SanteBien contient un mini-modèle IA frugal qui classe les questions santé sans donner de diagnostic. Nous avons suivi la démarche du cours : mesurer une baseline FP32, appliquer une optimisation INT8, puis comparer avant/après. Le modèle passe de 600 bytes à 154 bytes, soit 74,33 % de réduction. L'accuracy reste à 100 % sur notre échantillon et l'accord FP32/INT8 est de 100 %, donc la quantification ne crée pas de perte de qualité observée. La latence augmente de 29,74 % sur ce mini-modèle pur Python, mais reste très sous le seuil de 0,1 ms. Enfin, ces seuils sont automatisés dans la Green CI : si compression, accuracy, accord, taille, latence ou CO₂ régressent, le pipeline échoue.
